# PROJECT DISCOVERY

AI ตัวแรกของโครงการต้องถามคำถามเท่าที่จำเป็นเพื่อสร้าง Project Context โดยไม่ทำให้เจ้าของงานต้องออกแบบระบบเอกสารเอง

## ชุดคำถามขั้นต่ำ

### A. ตัวตนและเป้าหมาย

1. ชื่อโครงการและชื่อเรียกมาตรฐานคืออะไร
2. ปัญหาที่ต้องแก้คืออะไร
3. ใครเป็นผู้ใช้งานและผู้อนุมัติ
4. ผลลัพธ์สุดท้ายที่ถือว่าสำเร็จคืออะไร

### B. ขอบเขต

1. สิ่งใดอยู่ใน Scope
2. สิ่งใดไม่อยู่ใน Scope
3. มีโครงการหรือข้อมูลใดที่ห้ามปะปน
4. มีข้อห้ามด้านธุรกิจ กฎหมาย ความปลอดภัย หรือข้อมูลส่วนตัวหรือไม่

### C. สถานะเริ่มต้น

1. มีไฟล์ โค้ด หรือ Approved baseline เดิมหรือไม่
2. เวอร์ชันล่าสุดคืออะไรและสถานะใด
3. อะไรผ่าน QA แล้ว อะไรยังไม่ทดสอบ
4. เคยลองหรือแก้ปัญหาอะไรไปแล้ว

### D. วิธีทำงาน

1. ต้องการให้ AI ทำงานอิสระแค่ไหน
2. การเปลี่ยนแปลงใดต้องถามก่อน
3. รูปแบบ Version และ Approval ที่ต้องการ
4. ต้องการ checkpoint เมื่อใดบ้างนอกเหนือจากกฎกลาง

### E. งานแรก

1. งานถัดไปคืออะไร
2. Acceptance Criteria คืออะไร
3. สิ่งใดห้ามเปลี่ยนในรอบนี้
4. ใครเป็นผู้ตรวจรับและตรวจอย่างไร

## ผลลัพธ์หลัง Discovery

AI ต้องสร้างอย่างน้อย:

- `START_HERE.md`
- `MASTER_CONTEXT.md`
- `CURRENT_STATE.md`
- `PROJECT_RULES.md`
- `VERSION_CONTROL_RULES.md`
- `ROADMAP.md`
- `DECISION_LOG.md`
- `TASK_HISTORY.md`
- Task ปัจจุบันพร้อม Scope และ Acceptance Criteria

ข้อมูลที่ยังไม่ทราบให้เขียน `Pending` หรือ `Unknown` ห้ามเดาเติม
