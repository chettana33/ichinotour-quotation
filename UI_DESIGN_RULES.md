# UI DESIGN RULES

## หลักที่ต้องรักษา

1. หน้าแก้ไขอยู่ด้านซ้ายและ Preview เอกสารอยู่ด้านขวาบนจอ Desktop
2. เอกสารต้องพิมพ์และ Save PDF ได้โดยไม่แสดงส่วนควบคุม
3. สีหลัก CI ปัจจุบันเป็นน้ำเงินและฟ้า พร้อม Accent ที่ผู้ใช้ปรับได้
4. ต้องรักษาความอ่านง่ายของข้อความไทยและอังกฤษ
5. หมายเหตุรายการต้องแสดง label `หมายเหตุรายการ:` และรองรับหลายบรรทัด
6. Preview ที่แก้ข้อความได้ต้องไม่ทำให้ข้อมูลหลักสูญหายโดยไม่แจ้งผู้ใช้
7. การลบและการอัปเดตข้อมูลต้องมีข้อความยืนยันที่ชัดเจน

## พฤติกรรม Record Manager ปัจจุบัน

- Search: เลข Quotation, ลูกค้า, ชื่อทริป
- Filter: ปี, สถานะ, ปักหมุด
- Sort: แก้ไขล่าสุด, สร้างล่าสุด, ลูกค้า, เลข Quotation, วันเดินทาง
- Pagination: 20 รายการต่อหน้า
- Status: Draft, Sent, Confirmed, Cancelled

## กฎการเปลี่ยน UI

- ห้ามเปลี่ยน CI หรือ layout หลักนอก Scope
- ต้องตรวจทั้งหน้าจอและ Print Preview เมื่อแก้ CSS/HTML ที่เกี่ยวข้อง
- ต้องรักษาฟังก์ชัน keyboard/input และข้อความเตือนเดิมที่ไม่ได้อยู่ใน Scope
